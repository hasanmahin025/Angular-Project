C Constants

Constants in C are values that do not change during the execution of a program.
They are useful when you want to assign meaningful names to fixed values,
improving readability and maintainability of your code.

1. Why Use Constants?

  • Prevent accidental changes to fixed values.
  • Improve code readability by replacing "magic numbers" with meaningful names.
  • Facilitate easier updates—change the value in one place only.
  • Enforce program logic constraints (e.g., `MAX_USERS` should not change).

2. Types of Constants

  • Integer constants: Whole numbers, e.g., 100, -23
  • Floating-point constants: Decimal numbers, e.g., 3.14, 2.0e5
  • Character constants: Single characters enclosed in single quotes, e.g., 'A', '7'
  • String constants: Sequence of characters enclosed in double quotes, e.g., "Hello"

3. Defining Constants

A. Using #define (Macro Constants)

Defines a symbolic name that gets replaced before compilation. No type checking.

#define PI 3.14159
#define MAX_USERS 100

B. Using const Keyword

Declares a typed variable whose value cannot be modified after initialization. Compiler enforces immutability.

const int MAX_SCORE = 100;
const float TAX_RATE = 0.18;
const char NEWLINE = '\n';

4. Difference Between #define and const

Aspect #define const
Data type No type Has a data type
Scope Global (file-wide) Local or global
Memory allocation No memory reserved Memory is allocated
Debugging Harder to debug Visible in debugger
Compiler checks No type checking Yes, strictly checked

5. Literal Constants

These are raw values used directly in code, like:

  • 42 – integer literal
  • 3.14 – floating-point literal
  • 'A' – character literal
  • "World" – string literal

6. Example Program

#include <stdio.h>

#define PI 3.14159

int main() {  
    const int RADIUS = 5;
    float area = PI * RADIUS * RADIUS;
    printf("Area of Circle: %.2f\n", area);
    return 0;
  }

7. Notes and Best Practices

  • Use const when you need type safety and scoped constants.
  • Use #define for compile-time symbolic values or for conditional compilation.
  • Avoid magic numbers in code; replace them with named constants.
  • Declare const variables as static inside functions to limit scope and reduce memory overhead.
